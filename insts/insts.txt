
GROW Admin Dashboard – Full Feature & Requirements Document
Version 1.0 — December 2025

1. Overview
The GROW Admin Dashboard is the central control system for managing the entire GROW platform, including kiosk owners, kiosks, workers, customers, transactions, dues, redemptions, and app-wide settings.
It allows administrators to:
Oversee platform growth and performance
Manage verification processes
Control all user accounts
Monitor financial flows (dues, commissions, redemptions)
Ensure system integrity and prevent abuse
Configure platform-wide rules and settings
This dashboard must be user-friendly, data-driven, and built with role-based access control.

2. Primary Admin Personas
2.1 Super Admin
Full access to the entire dashboard
Can change global settings, fees, limits
Can manage other admin accounts
2.2 Admin
Can verify users, approve transactions, manage kiosks
Cannot change global fees or app-wide rules
2.3 Viewer
View-only access for auditing or support teams

18. Admin Team Management (Role-Based Model)

Admin Roles & Permissions

1. Owner (Super Admin)

Full System Access

The Super Admin has unrestricted access to every feature in the GROW Admin Dashboard. This role is responsible for platform-wide configuration, governance, and strategic oversight.

Capabilities include:

Full access to all dashboard modules and data

Modify global system settings (limits, fees, commissions, rules)

Create, edit, and remove admin accounts

Override or update any owner, kiosk, worker, or customer data

Approve or reject verifications, transactions, dues, and redemptions

Access to advanced analytics, security logs, and system configurations

Perform any operation related to financial or operational controls


This role is typically reserved for the founders or top-level management.



2. Editor (Admin)

Operational Admin – No Access to Global Configuration

The Editor manages the day-to-day operational tasks of the platform but is restricted from modifying sensitive system-wide parameters.

Capabilities include:

Review and accept owner, worker, or kiosk invitations

Approve or reject redemption requests

Edit and update kiosk dues after verification

Freeze or ban owners, kiosks, workers, or customers

Monitor transactions and activity

Access most analytics and reports

Manage verification workflows


Restrictions:

Cannot change any global rules

Cannot adjust system limits (transaction limits, goal limits, etc.)

Cannot modify fees, commissions, or financial structures

Cannot manage admin team roles

Cannot alter platform-wide configurations


This role is designed for operations staff, support teams, and field managers.



3. Viewer (Read-Only)

View Access Only

The Viewer can browse the dashboard and monitor activity without making any changes or performing any actions.

Capabilities include:

View all pages, data, and analytics

Access transaction reports

Access user and kiosk profiles

Monitor system activity


Restrictions:

Cannot edit, approve, reject, freeze, or update anything

Cannot access system settings

Cannot manage admin accounts


This role is ideal for auditors, analysts, and support agents who need visibility but no control.
